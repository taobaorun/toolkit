02

关键决策

纯原生 HTML/CSS/JS，零依赖
Chrome 扩展不需要打包工具。原生实现 ~1200 行代码即可覆盖全部 4 个模块，引入框架反而增加构建和调试成本。
Manifest V3 + sidePanel API
使用 Chrome 新架构的 sidePanel（非已废弃的 browser_action popup），获得持久化侧边栏，关闭后状态保留。
Markdown 用 marked 库渲染
用户指定。marked 成熟可靠，支持完整 GFM 语法（表格、task list、代码高亮），内联 ~35KB。
JSON diff 逐行 +− 标注
左右两侧逐行对齐，用 hash set 标记不同行，标注 +（新增）和 −（删除）。简单高效，不需要 LCS 算法。
AI Chat 不含 LLM API 调用
快捷提示词作为预设文本填入输入框。对话界面完整可用（用户手动输入），后续可接入 API 但不在本次实现范围。

架构布局

侧边栏固定宽度，垂直三段式布局。每个模块按需渲染不同内容区域。

侧边栏布局：Header → Tab Bar → Content Area（JSON Editor 模式示意，树形视图可折叠展开）
