Markdown 模块自动识别并渲染当前页内容

03

关键决策

读取页面用 chrome.scripting.executeScript 而非 fetch(tab.url)已选
理由：executeScript 能同时拿到"是否原始 markdown（单一 <pre> / text/plain）"的判断和渲染后可见文本；fetch 对普通网页只能拿到 HTML 源码，无法满足"回退抓取正文"的需求。代价是需新增 scripting 权限（注入目标已由现有 <all_urls> 覆盖）。
自动加载采用"空/示例才覆盖"的保护策略已选
理由：纯"每次打开都自动覆盖"会冲掉用户正在编辑的草稿；纯"只按钮"又不够"自动"。折中：render 时自动尝试一次，但只在源码框为空或等于初始示例时写入。这是比全自动更简单且更安全的方案。
识别信号 = 内容结构 + URL 后缀双重判断已选
理由：单看 document.contentType 会漏掉部分被包成 <pre> 的原始页；单看 URL 后缀会漏掉无 .md 后缀的 raw 链接。两者结合把误判降到最低。
