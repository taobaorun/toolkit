# 技术债务追踪

记录已知的技术债务、优先级和计划解决方案。

## 技术债务定义

技术债务是指为了短期目标而采取的非最优解决方案所产生的未来成本。不是所有技术债务都是坏的，关键是：

1. **有意识的**：知道自己在产生债务
2. **有计划的**：知道何时偿还
3. **有记录的**：不会遗忘

## 如何记录新的技术债务

当产生技术债务时，按以下格式记录：

```markdown
| TD-NNN | <简短描述> | <影响评估> | <计划时间或待定> | <如有 ExecPlan 则链接> |
```

评估标准：
- **高优先级**：影响用户体验、安全性或可维护性
- **中优先级**：影响开发效率或代码质量
- **低优先级**：小问题，可以在方便时处理
