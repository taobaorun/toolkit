# Issues 索引

本目录用于记录项目中发现的 Bug 和问题。

## 使用方法

发现新 Bug 时：

1. 复制 `TEMPLATE.md` 模板
2. 重命名为 `<日期>-<简短描述>.md`
3. 填写详细信息
4. 提交到版本控制

**修复 Bug 后，必须添加对应的测试用例**，并在模板的「测试用例」部分记录测试信息。

## Bug 严重程度定义

| 级别 | 描述 |
|------|------|
| **Critical** | 系统崩溃、数据丢失、严重功能失效 |
| **High** | 主要功能不可用，影响大量用户 |
| **Medium** | 功能部分受损，但有 workaround |
| **Low** | 轻微问题，不影响功能使用 |
