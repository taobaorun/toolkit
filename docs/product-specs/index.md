# 产品规格索引

本目录存放工具的功能需求和规格说明。

**本目录的具体规格文档使用 HTML 格式**（Tier 3 HTML native，激活 `html-artifact` skill 编写）。本索引文件保持 Markdown，方便 Agent 快速扫描。

## 项目愿景

Toolkit 是一套独立小工具的集合——每个工具解决一个具体问题，简单直接，即开即用。

## 优先级定义

- **P0**：核心功能，必须完成
- **P1**：重要功能，对用户体验有显著影响
- **P2**：期望功能，提升用户体验
- **P3**：锦上添花，有时间再做

## 如何编写工具规格

每个工具规格是一份 HTML 文档（激活 `html-artifact` skill 的 report 场景）。结构上应覆盖：

1. **工具概述**：这个工具是什么，解决什么问题
2. **使用场景**：典型使用场景和用户故事
3. **功能清单**：具体的功能和验收标准
4. **技术考量**：实现语言、依赖、限制
5. **未来计划**：可能的扩展方向

写完后在本索引文件追加一行。
